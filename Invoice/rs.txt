## TASK 1: Hỗ trợ tạo hóa đơn KHÔNG chọn khách hàng (Khách Lẻ)

### Mục tiêu business:
Hiện tại hệ thống đang yêu cầu phải chọn khách hàng trước khi tạo hóa đơn.
Cần thay đổi để:

1. Cho phép người dùng tạo hóa đơn ngay cả khi KHÔNG chọn bất kỳ khách hàng nào trong danh sách.
2. Nếu không chọn khách hàng:
   - Vẫn cho phép chuyển sang trang chọn sản phẩm bình thường.
   - Khi mở trang chọn sản phẩm, hệ thống phải tự động sử dụng giá bán mặc định là cột `PriceOdd`.
3. Khi lưu hóa đơn vào database trong trường hợp không chọn khách hàng:
   - `invoices.customer_id = NULL`
   - customer name dùng để hiển thị / lưu ngữ cảnh là `"Khách Lẻ"`
   - Vì không có khách hàng thực, nên KHÔNG có `customer_id`
4. Không làm hỏng luồng cũ:
   - Nếu có chọn khách hàng thật thì hệ thống vẫn hoạt động như hiện tại.
   - Logic giá theo khách hàng hiện tại (nếu đang có phân loại giá) phải vẫn giữ nguyên cho khách hàng đã chọn.

### Yêu cầu implementation:
- Kiểm tra và refactor logic validate ở CreateInvoicePage / CreateInvoiceViewModel nếu hiện tại đang chặn khi chưa chọn customer.
- Khi navigate sang ProductSelectionPage:
  - Truyền đầy đủ ngữ cảnh để biết:
    - Có customer thật hay không
    - Nếu không có customer thì đây là chế độ "Khách Lẻ"
- Trong ProductSelectionViewModel / ProductSelectionPage:
  - Nếu là "Khách Lẻ", mặc định giá bán phải map sang `PriceOdd`
  - Không được phụ thuộc vào customerId trong case này
- Khi lưu invoice:
  - `invoices.customer_id = NULL`
  - `invoice_details.customer = 'Khách Lẻ'` nếu không chọn khách
  - Các bản ghi inventory transaction vẫn phải được tạo bình thường và liên kết `invoice_id` đúng
- Nếu code hiện tại đang dùng selected customer null-check để chặn toàn bộ flow, hãy thay bằng logic phân nhánh:
  - `SelectedCustomer != null` => khách hàng thật
  - `SelectedCustomer == null` => khách lẻ

### Database schema (phải tuân thủ đúng):
1.
create table public.invoices (
  invoice_id text not null,
  customer_id bigint null,
  created_date date null default CURRENT_DATE,
  total bigint null default 0,
  constraint invoices_pkey primary key (invoice_id),
  constraint invoices_customer_id_fkey foreign KEY (customer_id) references customers (customer_id) on delete set null
) TABLESPACE pg_default;

2.
create table public.invoice_details (
  invoice_id text not null,
  product_id bigint not null,
  product_name text null,
  sell_price bigint null,
  amount integer null,
  line_total bigint GENERATED ALWAYS as ((sell_price * amount)) STORED null,
  customer text null,
  note text null,
  constraint invoice_details_pkey primary key (invoice_id, product_id),
  constraint invoice_details_invoice_id_fkey foreign KEY (invoice_id) references invoices (invoice_id) on delete CASCADE
) TABLESPACE pg_default;

3.
create table public.inventorytransactions (
  transaction_id bigint generated by default as identity not null,
  product_id bigint null,
  invoice_id text null,
  date date not null default CURRENT_DATE,
  amount integer null,
  note text null,
  source_type text null,
  product_name text null,
  type text null,
  constraint inventorytransactions_pkey primary key (transaction_id),
  constraint inventorytransactions_invoice_id_fkey foreign KEY (invoice_id) references invoices (invoice_id) on delete CASCADE
) TABLESPACE pg_default;

### Kỳ vọng behavior:
- Có chọn khách => giữ nguyên flow cũ
- Không chọn khách => vẫn tạo được hóa đơn
- Giá mặc định cho khách lẻ = `PriceOdd`
- Lưu DB hợp lệ với `customer_id = NULL`
- Không phát sinh lỗi null reference ở bất kỳ bước nào

## TASK 2: Tạo hàm xử lý chuỗi nâng cao cho tên khách hàng

### Mục tiêu:
Tạo helper function để normalize tên khách hàng theo chuẩn:
- Viết hoa chữ cái đầu của từng từ
- Các ký tự còn lại chuyển về lowercase đúng Unicode
- Loại bỏ khoảng trắng thừa ở đầu, cuối, và giữa nhiều khoảng trắng liên tiếp chỉ còn 1 space
- Hỗ trợ tiếng Việt có dấu đúng cách

### Ví dụ:
Input:
"    NgUyễN     ThUẬn    "

Output:
"Nguyễn Thuận"

### Vị trí cần đặt:
@Invoice/Helpers/StringHelper.cs

### Yêu cầu implementation:
- Nếu file chưa có class phù hợp thì tạo static helper class
- Tạo method rõ nghĩa, ví dụ:
  - `NormalizeVietnameseName(string input)`
  hoặc tên tương đương dễ hiểu
- Method phải:
  1. Null-safe
  2. Trim đầu/cuối
  3. Gộp nhiều space thành 1
  4. Chuyển từng từ về Title Case phù hợp cho tiếng Việt Unicode
- Hạn chế side effects
- Dễ tái sử dụng

### Yêu cầu áp dụng:
- Tích hợp hàm này vào luồng tạo / chỉnh sửa / chọn khách hàng nếu có nơi nhập tên khách
- Ưu tiên áp dụng tại:
  - CustomersViewModel
  - CustomersPage
  - CreateInvoiceViewModel
  nếu các nơi này đang nhận hoặc hiển thị tên khách hàng từ input người dùng
