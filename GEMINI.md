# Invoice Project Overview

This document provides a comprehensive overview of the `Invoice` project architecture, technology stack, and development guidelines.

## 🚀 Tech Stack

- **Framework**: [WinUI 3](https://docs.microsoft.com/windows/apps/winui/winui3/) (Windows App SDK 1.8+)
- **Runtime**: .NET 8.0 (Windows 10/11 x64)
- **Architecture**: MVVM (Model-View-ViewModel)
- **Backend-as-a-Service**: [Supabase](https://supabase.com/) (PostgreSQL, Auth, Realtime)
- **Dependency Injection**: Microsoft.Extensions.Hosting / DependencyInjection
- **UI Components**: 
  - [CommunityToolkit.Mvvm](https://github.com/CommunityToolkit/dotnet) (ObservableObject, RelayCommand)
  - [CommunityToolkit.WinUI](https://github.com/CommunityToolkit/WindowsCommunityToolkit) (DataGrid, Behaviors)
  - [WinUIEx](https://dotmorten.github.io/WinUIEx/) (Window management)
  - [LiveCharts2](https://livecharts.dev/) (Data visualization)
- **Reporting & PDF**: [QuestPDF](https://www.questpdf.com/)
- **Data Access**: [Dapper](https://github.com/DapperLib/Dapper) & [Npgsql](https://www.npgsql.org/) (for direct DB interaction when needed)
- **Email**: [MailKit](https://github.com/jstedfast/MailKit) (SMTP)

## 🏗️ Project Architecture

The solution is divided into two main projects:

### 1. `Invoice` (UI Layer)
- **Views/**: XAML pages and code-behind.
- **ViewModels/**: Business logic for the UI, using `ObservableProperty` and `RelayCommand`.
- **Services/**: UI-specific services (Navigation, Activation, Theme, PDF Generation).
- **Styles/**: XAML ResourceDictionaries for consistent UI/UX.
- **Strings/**: Localization files (`Resources.resw`), primary language is Vietnamese (`vi-VN`).

### 2. `Invoice.Core` (Logic Layer)
- **Models/**: Data entities (Invoices, Customers, Products, etc.).
- **Services/**: Core business logic and data access.
- **Contracts/**: Interfaces for services and messaging (Pub/Sub).
- **Helpers/**: Common utilities (JSON, String formatting).

## 📡 Data & Backend

- **Supabase**: Serves as the primary database and backend.
  - `SupabaseDataService` handles CRUD operations and caching.
  - Real-time updates are implemented via Supabase Realtime for inventory/product changes.
  - PostgreSQL functions (RPC) are used for complex transactions (e.g., `create_full_invoice`).
- **Caching**: `InMemoryCache` is used in `SupabaseDataService` to reduce API calls for static/slow-changing data like Materials and Frames.

## 📄 Reporting & PDF

- **QuestPDF**: Used to generate professional invoices and reports.
- **Templates**:
  - `GenerateOfficial`: Standard invoice with logo and detailed pricing.
  - `GenerateTemp`: Quick estimate/temporary invoice.
  - `GenerateReport`: Sales and inventory reports.

## 🛠️ Development Guidelines

- **MVVM**: Always keep business logic in ViewModels. Use `App.GetService<T>()` for Dependency Injection.
- **Navigation**: Use `INavigationService` for page-to-page navigation. Parameters are passed via `NavigateTo`.
- **Configuration**: All secrets and settings (Supabase, Email) are stored in `appsettings.json`.
- **Naming Convention**:
  - Services: `ISomethingService` (Contract) / `SomethingService` (Implementation).
  - ViewModels: `[PageName]ViewModel`.
  - Commands: `[Action]Command` (e.g., `SaveInvoiceCommand`).

## 🌐 Localization
- Primary Language: **Vietnamese (vi-VN)**.
- Secondary Language: **English (en-us)**.
- Localization is handled via `ResourceExtensions` and `resw` files.
